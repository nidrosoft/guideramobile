## Quick Trip - Add-ons Logic

### When user can preview WITHOUT booking:
- User completes steps 1-4
- AI generates itinerary
- User can view full plan
- User can save as "Planned" state
- No actual bookings made

### When user wants to add bookings:
- From Review step, user taps "+ Add Hotel" (or Flight/Car/Experience)
- Opens respective booking flow as a modal
- On completion, booking is linked to the plan
- User returns to Review step with booking attached
- Can add multiple bookings

### Preview Restriction Logic:
```
IF user adds Hotel OR Flight:
  â†’ Plan becomes "Booking Required" mode
  â†’ Must complete booking to confirm
  â†’ Can still save as draft
  
IF user has NO bookings:
  â†’ Can preview full itinerary freely
  â†’ Can confirm as "Planned" trip
  â†’ No payment required
```
