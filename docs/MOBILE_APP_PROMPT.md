## 1. NEW DATABASE TABLES (already created ‚Äî do NOT create migrations)

### `affiliate_click_tracking`
Tracks every deal link click with affiliate attribution. Key columns:
- `id` (uuid, auto) ‚Äî Primary key
- `tracking_id` (text, auto-generated) ‚Äî Unique ID format: `gdr_<uuid_no_dashes>` (auto-generated by DB)
- `user_id` (uuid, required) ‚Äî FK ‚Üí profiles.id
- `deal_cache_id` (uuid, nullable) ‚Äî FK ‚Üí deal_cache.id
- `deal_match_id` (uuid, nullable) ‚Äî FK ‚Üí user_deal_matches.id
- `deal_type` (text, required) ‚Äî "flight", "hotel", "experience", "package"
- `provider` (text, required) ‚Äî e.g. "kiwi", "booking", "viator"
- `route_key` (text, nullable) ‚Äî e.g. "SAN-CDG"
- `price_amount` (numeric, nullable) ‚Äî Deal price at click time
- `price_currency` (text, default 'USD')
- `deal_title` (text, nullable) ‚Äî Display title of the deal
- `affiliate_config_id` (uuid, nullable) ‚Äî FK ‚Üí affiliate_config.id
- `affiliate_url` (text, required) ‚Äî The full affiliate URL the user was sent to
- `commission_rate` (numeric, nullable) ‚Äî Rate at time of click (from affiliate_config)
- `estimated_commission` (numeric, nullable) ‚Äî price_amount √ó commission_rate
- `source` (text, required, default 'app') ‚Äî One of: "app", "push", "email", "in_app_deal", "campaign"
- `campaign_id` (text, nullable) ‚Äî If from a campaign notification
- `notification_id` (uuid, nullable) ‚Äî FK ‚Üí deal_notifications.id if triggered from a notification
- `clicked_at` (timestamptz, default now())
- `device_type` (text, nullable) ‚Äî "ios" or "android"
- `app_version` (text, nullable) ‚Äî App version string
- `ip_address` (text, nullable) ‚Äî User's IP (if available)

### `deal_campaigns` (read-only from mobile ‚Äî admin creates these)
Campaigns created by the admin panel. The mobile app should:
- Receive push notifications triggered by campaigns
- Include `campaign_id` in click tracking when a user clicks from a campaign notification

### `deal_campaign_sends` (written by edge functions, read by mobile)
Per-user campaign delivery tracking. Columns relevant to mobile:
- `campaign_id`, `user_id`, `channel`, `status`
- `opened_at` ‚Äî **Mobile app should UPDATE this** when user opens a campaign notification
- `clicked_at` ‚Äî **Mobile app should UPDATE this** when user clicks through from a campaign notification

### B. Wrap ALL External Deal Links
Every place in the app where the user taps a deal and is sent to an external URL (Kiwi, Booking.com, Viator, Rentalcars, etc.), you MUST:
1. Call `trackAffiliateClick()` BEFORE opening the URL
2. Open the affiliate URL in the in-app browser or system browser
3. Pass `source` correctly:
   - `"app"` ‚Äî User browsing deals in the deals tab
   - `"push"` ‚Äî User tapped a push notification
   - `"campaign"` ‚Äî User tapped a campaign-triggered notification
   - `"in_app_deal"` ‚Äî User tapped a deal from the in-app deal feed/card

### C. Search Session Tracking
When the user performs any search (flight, hotel, experience, package):
1. Create a row in `search_sessions` at search START
2. Update the row with `results_count`, `lowest_price`, `highest_price` when results arrive
3. Update `selected_result_index` when user taps a result
4. Update `session_duration_ms` when user leaves the results screen

### D. Deal Match Engagement
When showing deal matches to the user:
1. When a deal card enters the viewport, update `user_deal_matches` SET `viewed = true` WHERE `id = <match_id>`
2. When user taps a deal card, update `user_deal_matches` SET `clicked = true` WHERE `id = <match_id>`

### E. Push Notification Handling for Campaigns
When the app receives a push notification:
1. Check if the notification payload includes `campaign_id`
2. If yes, when the user **opens** the notification, UPDATE `deal_campaign_sends` SET `opened_at = now()` WHERE `campaign_id = <id>` AND `user_id = <user_id>`
3. When the user **clicks through** to the deal, UPDATE `deal_campaign_sends` SET `clicked_at = now()` WHERE `campaign_id = <id>` AND `user_id = <user_id>`
4. Include `campaign_id` in the `trackAffiliateClick()` call

### F. Device & Version Metadata
For all tracking calls, include:
- `device_type`: "ios" or "android" (from `Platform.OS`)
- `app_version`: Your app version string (from app.json or Constants)

## 4. AFFILIATE CONFIG LOOKUP

The `affiliate_config` table has 6 providers configured. When building affiliate URLs:

```sql
SELECT id, provider, display_name, base_url, url_template, commission_rate, commission_model
FROM affiliate_config
WHERE is_active = true AND provider = '<provider_name>';
```

Use the `url_template` to construct affiliate URLs with proper tracking parameters. The `commission_rate` should be stored in each click tracking record for historical accuracy.

**Cache this config** on app startup or on first deal view ‚Äî it rarely changes.

## 5. RLS POLICIES

The `affiliate_click_tracking` table has RLS enabled. Users can only INSERT their own rows:
- INSERT: `auth.uid() = user_id`
- SELECT: `auth.uid() = user_id` (users can see their own clicks)

The `deal_campaign_sends` table:
- SELECT: `auth.uid() = user_id`
- UPDATE: `auth.uid() = user_id` (for opened_at, clicked_at)

The `search_sessions` table:
- INSERT: `auth.uid() = user_id`
- SELECT: `auth.uid() = user_id`

Make sure the Supabase client uses the authenticated user's JWT for all these operations.
